================================================================================
FIX SUMMARY
================================================================================

PROBLEMS FIXED:
================================================================================

1. Phone being sent as null
   CAUSE: JSON serialization wasn't properly sending body
   SOLUTION: 
   - Refactored ApiTestService.callApi() method
   - Now properly serializes JSON with phone and password fields
   - Added charset=utf-8 header for proper encoding

2. Body not editable
   CAUSE: TextArea was static
   SOLUTION:
   - Set bodyTextArea.setEditable(true)
   - Users can now modify request body before testing
   - Auto-generates from test case if empty

3. Endpoint not customizable
   CAUSE: Hardcoded endpoint path
   SOLUTION:
   - Set baseUrlField.setEditable(true)
   - Extracts /api/v1/... from full URL or uses as-is
   - Allows testing different endpoints

================================================================================
CODE CHANGES
================================================================================

FILE: ApiTestService.java
============================

OLD: Direct POST with hardcoded /api/v1/signup
NEW: Two methods:
  1. callSignupApi(phone, password) - builds JSON and calls callApi()
  2. callApi(endpoint, jsonBody) - generic API caller with custom endpoint/body

BENEFITS:
- Proper JSON serialization
- Support for custom endpoints
- Support for custom body content
- Better error handling

FILE: TestcaseController.java
=============================

OLD: callActualApi() only used test case data
NEW: callActualApi() now:
  1. Reads body from bodyTextArea
  2. Falls back to auto-generated body if empty
  3. Reads endpoint from baseUrlField
  4. Extracts path from full URL if needed
  5. Calls apiTestService.callApi(endpoint, body)

ALSO:
- handleApiSelection() now sets both fields as editable
- bodyTextArea.setWrapText(true) for better display

BENEFITS:
- Flexible body content
- Custom endpoint support
- Fallback to defaults if fields are empty
- Better user control

================================================================================
USAGE FLOW
================================================================================

1. User selects API from tree
   → bodyTextArea loads sample JSON
   → baseUrlField loads endpoint
   → Both are now editable

2. User can modify:
   → URL in baseUrlField (http://... or /api/...)
   → Body in bodyTextArea (any valid JSON)

3. User clicks "Run All"
   → For each test:
      - Read custom body (if provided)
      - Read custom endpoint (if provided)
      - Call apiTestService.callApi(endpoint, body)
      - Send as HTTP POST with proper JSON content-type
      - Validate response code

4. Backend receives:
   → POST request to specified endpoint
   → Proper JSON body with phone and password fields
   → Correct headers (Content-Type: application/json, charset=utf-8)
